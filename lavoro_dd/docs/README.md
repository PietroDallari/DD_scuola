# docs

Cartella per i **report finali prodotti da console brother** (documenti di sintesi, note bilaterali per Nazareno, versioni pronte da condividere).

A differenza di `lavoro_dd/analisi/` — che contiene i nostri run, driver, CSV e riscontri operativi datati — qui vanno i **documenti finiti**: il testo che si legge, non i dati grezzi da cui nasce.

Convenzione suggerita: un file per documento, nome descrittivo con data (es. `report_ciclo_unico_YYYY-MM-DD.md`). I dati e le analisi a supporto restano linkabili da `../analisi/`.
